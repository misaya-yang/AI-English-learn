## Critical (ship blocker)

### [TODAY-01] WordWorkbench card is entirely dark-heritage on a light-first page
**Page/Component:** `src/pages/dashboard/TodayPage.tsx` — `WordWorkbench` component (lines 73–258), `ConfettiCelebration`, `CircularProgress`
**Problem:** Every inner surface of the flashcard (`text-white`, `text-white/42`, `bg-white/[0.03]`, `border-white/[0.08]`, emerald-on-dark badges) is hardcoded in dark-glass tokens. On a light desktop the cards render as near-invisible white-on-white. The CircularProgress SVG also hardcodes `fill: 'rgba(255,255,255,0.9)'` and `stroke="rgba(255,255,255,0.08)"` which disappear on white backgrounds. The flash card back ("Word detail") header uses `text-white` unconditionally. This is the product's *primary learning surface* — it breaks for any user on the system-light or forced-light theme.
**Impact:** Every light-mode user sees a broken primary learning surface the first session. Likely the #1 churn driver for new signups who land on the default light theme.
**Suggestion:** Audit all classes in `WordWorkbench` and replace with semantic tokens (`text-foreground`, `text-muted-foreground`, `bg-card`, `border-border`). Replace hardcoded SVG fill/stroke with CSS custom properties or Tailwind tokens. The light-rail sections already use `dark:text-white/42` double-class patterns correctly — extend that pattern to the card internals.

### [TODAY-02] TodayPage flashcard nav dots have no accessible labels
**Page/Component:** `src/pages/dashboard/TodayPage.tsx` lines 828–850
**Problem:** The dot-navigation buttons that jump between today's words have no `aria-label` (the `title` attr is present but `title` alone does not satisfy WCAG 2.1 SC 4.1.2 — interactive controls need accessible names). With keyboard navigation focused on dot-3 the screen reader announces "button" with no word context.
**Impact:** Fails WCAG AA; enterprise B2B buyers in education (a target segment per the PRD) routinely require WCAG AA before procurement approval.
**Suggestion:** Add `aria-label={word.word}` (already in `title`). Also add `role="group"` with `aria-label="Word navigation"` to the wrapping `div`.

### [LEARNPATH-01] LearningPathPage: checking a lesson box does NOT navigate to any learning content — it only toggles a completion flag
**Page/Component:** `src/pages/dashboard/LearningPathPage.tsx` lines 252–285
**Problem:** Every lesson row is a `<button>` that calls `handleToggleLesson`. There is no routing, no modal, no content — just a checkbox toggle on a flat data model (`learningPathProgress` localStorage). The "Suggested next step" card (line 218) shows the lesson title but there is no `href` or action to start it. A user who clicks "Start" on a lesson path has nothing to do — they can only manually tick boxes.

**Suggestion:** Either (a) route each lesson to the relevant dashboard page (vocabulary lessons → `/dashboard/today`, grammar → `/dashboard/practice`, conversation → `/dashboard/chat`) by tagging each `LessonItem` with a `targetRoute`, or (b) surface an explicit "This is your learning map — click a skill to navigate" explanation. The immediate fix is adding `targetRoute` to the data model and making lesson buttons navigate instead of toggle.

### [PRICING-01] Pricing page shows a "Coming soon" notice and a `mailto:` link but the email domain is `vocabdaily.ai` — not registered; the Supabase project ref is visible in the copy-setup-guide

### [TODAY-03] No keyboard shortcut for flip — mobile users have no TTS auto-play on card advance
**Page/Component:** `src/pages/dashboard/TodayPage.tsx`
**Problem:** ReviewPage has keyboard shortcuts (Space = reveal, 1-4 = rate). TodayPage has none. There is no global key handler for `Space` to flip the current word card, nor for `←`/`→` to navigate. On mobile, TTS is not triggered automatically when advancing to the next word (the user must tap the Volume icon separately), adding friction to the most-used flow.

### [TODAY-04] Empty state when `activeBook` is null shows "AI help" CTA that requires an active book anyway
**Page/Component:** `src/pages/dashboard/TodayPage.tsx` lines 626–669
**Problem:** The empty state shown when `words.length === 0` offers "让 AI 帮我规划" (let AI help me plan) as a secondary CTA. ### [REVIEW-01] ReviewPage does not surface the AI memory-cue until after reveal, but the hint is only shown if `memoryTip || etymology` is present — no fallback for words without it

### [SETTINGS-01] SettingsPage uses `window.confirm()` for the destructive "Clear All Data" action
**Page/Component:** `src/pages/dashboard/SettingsPage.tsx` line 82
**Problem:** `if (confirm('Are you sure...'))` uses the browser's native blocking dialog, which (1) looks unprofessional and out of place in a polished SaaS, (2) cannot be styled or translated (the page has i18n support elsewhere), (3) is blocked by popup-blockers in some enterprise browser policies, and (4) has no 30-second undo grace period.
